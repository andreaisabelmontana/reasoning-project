Reasoning Project

Strip a game down to its essence — a set of states, the moves between them, and a goal — and "playing well" becomes "searching cleverly." This project studies automated reasoning over a game environment from first principles: how to represent a problem so a machine can think about it, and which search makes it find the answer fastest. I rebuilt it from scratch because this is the bedrock of how AI plans at all.

What it is

Reasoning as search over states

The central idea of classical AI is disarmingly simple: nearly any puzzle can be framed as a graph of states connected by actions, with a goal somewhere in it. Solving the puzzle is finding a path to that goal. This project takes a game environment and builds exactly that — a clean model of states, legal moves and goals — then turns different search algorithms loose on it to compare how they reason.

Doing it from scratch makes the trade-offs concrete. Breadth-first guarantees the shortest path but devours memory; depth-first is lean but can wander; A* uses a heuristic to head straight for the goal — if the heuristic is good. The same problem, solved well or badly depending on how the machine searches.

Architecture

How a solution is found

Every solver shares the same model and differs only in how it explores:

  1. Frame

    Define the start state, the goal test and the legal actions.

  2. Expand

    From the current state, generate its successor states.

  3. Choose

    The search strategy decides which state to explore next.

  4. Guide

    Informed search uses a heuristic to prioritise promising paths.

  5. Recover

    On reaching the goal, trace back the path that got there.

Reflection

What rebuilding it taught me

  • Representation comes first. How you model states and actions decides whether the search is easy or hopeless.
  • The algorithm is a trade-off. BFS, DFS and A* each win on a different axis — completeness, memory or speed.
  • A good heuristic is everything. A* is only as smart as its estimate; the right heuristic turns a huge search into a beeline.
  • Admissibility guarantees optimality. A heuristic that never overestimates is what lets A* promise the best path.
  • Search is reasoning. "Planning a route" and "playing a puzzle" are the same act — finding a path through a space of possibilities.
